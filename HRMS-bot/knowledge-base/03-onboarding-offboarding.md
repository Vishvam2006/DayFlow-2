# Dayflow HRMS - Employee Access and Lifecycle Guidance

## Employee Access Setup
Each employee profile in Dayflow HRMS includes:
- Name
- Email
- Phone number
- Job title
- Department
- Employee ID
- Salary structure

The registered phone number is important for WhatsApp bot verification. The number must be stored in E.164 format, for example `+91XXXXXXXXXX`.

## Employee Portal Structure
The employee dashboard currently includes:
- Dashboard
- My Tasks
- Attendance
- Leave
- Payslips
- Profile

The bot may guide employees to these sections when a dashboard action is more appropriate than a WhatsApp action.

## Admin Portal Structure
The admin dashboard currently includes:
- Dashboard
- Employees
- Departments
- Leave Requests
- Task Manager
- Payroll
- Analytics
- Office Network
- Profile

The bot may reference these areas when explaining how a request is processed, but it must not provide admin-only details to employees unless they are already present in the employee's allowed context.

## Employee Creation and Credentials
- Admins create employee accounts from the Employees section.
- A unique employee ID is generated by the backend.
- A generated password is shown once at employee creation time.
- Duplicate email addresses and duplicate phone numbers are not allowed.

## Offboarding and Account Changes
If an employee needs profile corrections, role updates, or access-related changes, the bot should direct them to HR or an authorized administrator. The bot should not promise direct edits to protected profile, salary, or admin-managed account details unless a supported workflow exists.
